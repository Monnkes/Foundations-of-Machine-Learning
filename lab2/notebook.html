Zbir danych - klasyfikacja danych numerycznych

Wykorzystamy zbir danych Codon usage z dziedziny bioinformatyki. Zosta on zaprezentowany w artykule:

Hallee, Logan, and Bohdan B. Khomtchouk. "Machine learning classifiers predict key genomic and evolutionary traits across the kingdoms of life." Scientific Reports 13.1 (2023): 2088. link

Autorzy wykonywali na nim wiele analiz, ale nas interesuje podstawowa - przewidywanie, z jakiego krlestwa (kingdom) pochodzi komrka, na podstawie rozkadu kodonw w jej sekwencjonowanym RNA. Analiza DNA oraz RNA jest podstawowym zadaniem bioinformatyki. Kodony (codons) to trjki nukleotydw, np. UGC, i wikszo zapisuje pewien aminokwas (z wyjtkiem trzech kodonw stopu), ktry jest wykorzystywany w syntezie biaek.

Problem jest taki, e w praktyce trzeba sekwencjonowa DNA/RNA z wielu komrek, wic trzeba je namnoy w laboratorium. Niestety nie da si uzyska idealnej czystoci, i zawsze jest szansa, e prbka zostanie zanieczyszczona, np. bakteriami. Wtedy oprcz interesujcych nas komrek (np. zwierzcych) otrzymamy inne, ktrych trzeba si pozby przed dalsz analiz. Tutaj wanie wchodzi ML - dokonamy klasyfikacji, z jakiego krlestwa (a waciwie domeny) pochodzi kod RNA, eby pomc w takich sytuacjach.

Dla uproszczenia, autorzy, zamiast klasyfikowa krlestwo, klasyfikuj domen) organizmu. S to jednostki o poziom wyej w systematyce taksonomicznej. Do tego dokadamy wirusy i bakteriofagi. Mamy zatem klasy:

  1. Archaea (archeony) - drobne jednokomrkowce, rdo wanych enzymw stosowanych w biologii molekularnej
  2. Bacteria (bakterie)
  3. Eukariota (eukarionty) - midzy innymi ludzie
  4. Viruses (wirusy) - nie s organizmami ywymi, wic nie s czci typowej taksonomii, ale s wane w badaniach mikrobiologicznych
  5. Bacteriophages (bakteriofagi) - rodzaj wirusw atakujcych tylko bakterie, uywane m.in. w badaniach nad mikrobiomem jelitowym

Szczegowy opis zbioru znajduje si na stronie UCI. W skrcie:

  1. Kingdom - krlestwo, z ktrego pochodzi komrka. Podzia jest tutaj do szczegowy, ale organizmy te mona pogrupowa w praktyce na bakterie (bacteria), wirusy (viruses) i eukarionty (eukariota).
  2. DNAtype - uywana w innych analizach w artykule.
  3. SpeciesID - numer gatunku.
  4. Ncodons - liczba kodonw zmierzona dla danej komrki.
  5. SpeciesName - nazwa gatunku.

Dalsze kolumny to znormalizowany uamek kodonw poszczeglnych typw w RNA danej komrki. W zwizku z tym, e dane s ju znormalizowane, nie ma potrzeby ich skalowania.

Jak wida, mamy nie tylko klasyfikacj wieloklasow, ale na dodatek klasyfikacj niezbalansowan. W takim przypadku trzeba uy odpowiednich metryk. Wzorujc si na artykule, uyjemy macro-averaged F1-score. Jest to proste rozwinicie F1-score na wiele klas.

Dla przypomnienia, w klasyfikacji binarnej, F1-score to rednia harmoniczna precyzji (precision) i czuoci (recall):

$$ F1 = \frac{2}{recall^{-1} + precision^{-1}} = 2 * \frac{precision * recall}{precision + recall} = \frac{2 * TP}{2 * TP + FP + FN} $$

Macro-averaging oznacza, e najpierw obliczamy F1-score dla kadej klasy, a pniej bierzemy ich redni. eby taka rednia bya wysoka, to musimy mie wysoki wynik na wszystkich 5 klasach, w tym 2 mniejszociowych, wic uwzgldnimy niezbalansowanie klas.

Approximate Nearest Neighbors (ANN)

Postaramy si teraz przyspieszy nasz klasyfikator, w zamian za odrobin dokadnoci, za pomoc algorytmu przyblionych najbliszych ssiadw, a konkretnie NN-Descent. Ten zosta wybrany gwnie ze wzgldu na atw w uyciu implementacj.

Twrcy Scikit-learn'a zdawali sobie spraw, e algorytmy znajdowania najbliszych ssiadw to aktywne pole rozwoju, dlatego stworzyli klas (i interfejs) KNeighborsTransformer, pozwalajc integrowa zewntrzne rozwizania. Dziaa ona zarwno do klasyfikacji / regresji kNN, jak i do innych algorytmw opartych o ssiedztwo, ktre buduj graf najbliszych ssiadw. W przypadku jego uycia trzeba poda metric="precomputed", bo obliczaniem odlegoci zajmuje si osobny algorytm.

Implementacja PyNNDescent korzysta wanie z tego rozwizania, co gwarantuje atwo uycia - klasa PyNNDescentTransformer dziedziczy po KNeighborsTransformer. Zobaczmy, jak to dziaa. Wykorzystamy funkcj make_pipeline(), ktra buduje obiekt Pipeline, ale nie wymaga podawania nazw kolejnych elementw.

Uwaga: PyNNDescent dla wydajnoci korzysta z Numby, czyli kompilatora JIT (Just-In-Time). Dziki temu dziaa to bardzo szybko, ale pierwszy import i wykonanie zajmuje sporo czasu. Wykonaj kod 2 razy - za drugim razem bdzie o wiele szybciej.

In[]:
from time import time


def benchmark_knn_and_ann(
    sklearn_knn,
    pynndescent_ann,
    X_train,
    X_test,
    y_train,
    y_test,
) -> None:
    # training
    start_time = time()
    sklearn_knn.fit(X_train, y_train)
    end_time = time()

    sklearn_knn_fit_time = end_time - start_time

    start_time = time()
    pynndescent_ann.fit(X_train, y_train)
    end_time = time()

    pynndescent_ann_fit_time = end_time - start_time

    # prediction

    start_time = time()
    y_pred_sklearn = sklearn_knn.predict(X_test)
    end_time = time()

    sklearn_knn_predict_time = end_time - start_time

    start_time = time()
    y_pred_pynndescent = pynndescent_ann.predict(X_test)
    end_time = time()

    pynndescent_ann_predict_time = end_time - start_time

    f1_knn = f1_score(y_test, y_pred_sklearn, average="macro")
    f1_ann = f1_score(y_test, y_pred_pynndescent, average="macro")

    print(f"Scikit-learn training time: {sklearn_knn_fit_time:.2f}")
    print(f"PyNNDescent training time: {pynndescent_ann_fit_time:.2f}")
    print()
    print(f"Scikit-learn prediction time: {sklearn_knn_predict_time:.2f}")
    print(f"PyNNDescent prediction time: {pynndescent_ann_predict_time:.2f}")
    print()
    print(f"Scikit-learn F1: {100 * f1_knn:.2f}%")
    print(f"PyNNDescent F1: {100 * f1_ann:.2f}%")

Zbir danych - klasyfikacja danych chemicznych

Teraz zajmiemy si klasyfikacj zbioru posiadajcego same zmienne binarne, do ktrych trzeba uy odpowiednich metryk. Konkretnie, bdzie to zbir HIV z benchmarku MoleculeNet, w ktrym na podstawie reprezentacji chemicznej molekuy trzeba przewidywa, czy jest ona inhibitorem wirusa HIV. Inhibitory to substancje przeciwne do katalizatorw - spowalniaj lub uniemoliwiaj jakie zjawisko, w naszym wypadku infekcj wirusa HIV. Przewidywanie wasnoci moleku to kluczowe zadanie w nowoczesnym projektowaniu lekw i bardzo wane zastosowanie ML w farmacji i chemii molekularnej.

Zbir jest ju podzielony na treningowy, walidacyjny i testowy w ramach projektu Open Graph Benchmark. Dane te mona traktowa jak klasyfikacj grafw molekularnych, np. z pomoc grafowych sieci neuronowych. Jednak podejciem klasycznym i dajcym czsto lepsze wyniki s fingerprinty molekularne (molecular fingerprints). S to deterministyczne algorytmy ekstrakcji cech z grafu, zamieniajce go na wektor. Jest ich duo, ale najpopularniejszy to ECFP (Extended Connectivity FingerPrint), wykorzystujcy informacj o subgrafach o maym promieniu (typowo 4). Domylnie skutkuje to cechami binarnymi, czy subgraf wystpi, czy nie w danej molekule.

Molekuy s typowo przechowywane w formacie SMILES strings. Graf molekularny mona odczyta z tego formatu, a potem przekaza taki zbir do fingerprintu. Wtedy zamieniamy problem na zwyk klasyfikacj tabelaryczn. Scikit-fingerprints zrobi to automatycznie, jeeli SMILES strings zostan przekazane do fingerprintu.

Jako e zbir jest binarny, to nie ma potrzeby adnego skalowania zmiennych. W przypadku zbioru HIV typow metryk jest Area Under Receiver Operating Characteristic (AUROC / ROC AUC).

Wyszukiwanie z pomoc najbliszych ssiadw

Naszym ostatnim zastosowaniem metod opartych o ssiedztwo bdzie wyszukiwanie przedmiotw. Konkretnie stworzymy wyszukiwark obrazw na podstawie wpisanego hasa.

Jest to zadanie multimodalne (multimodal), bo operujemy na dwch rnych modalnociach: obrazie i tekcie. Takie wyszukiwarki dziaaj nastpujco:

  1. W fazie treningu przedmioty (obrazy) s wektoryzowane sieci neuronow i przechowywane w indeksie.
  2. Uytkownik wpisuje tekst, ktry jest wektoryzowany za pomoc tej samej sieci neuronowej, dlatego musi by ona multimodalna.
  3. Wyszukujemy najbliszych ssiadw (obrazy-wektory) dla tekstu-wektora.

Najbardziej znanym modelem multimodalnym jest CLIP, stworzony przez OpenAI. Jego architektura i wagi s dostpne publicznie, w przeciwiestwie do nowych rozwiza tej firmy. Jest to model zasadniczo niewymagajcy fine-tuningu, dziaa bardzo dobrze w podstawowej formie dla wielu zada. Zosta te wykorzystany jako element modelu DALL-E 2.

Nie musisz szczegowo wiedzie, jak dziaa ten model, ale pewne podstawy s bardzo ciekawe i przydatne. Wysokopoziomowo architektur dobrze opisuje ten artyku. Dla zainteresowanych cay artyku, znacznie bardziej szczegowy. CLIP skada si z dwch enkoderw: dla tekstu i dla obrazu. S to zwyke pre-trenowane sieci, a konkretnie:

  1. Dla obrazu - Visual Transformer (ViT). Jest to architektura, ktra uywa transformerw i atencji do przetwarzania obrazw. Ide jest, eby poci obraz na ssiadujce kawaki (patches), i traktowa je jak sowa dla atencji.
  2. Dla tekstu - Transformer, z pewnymi modyfikacjami.

Kada z tych sieci wektoryzuje wejcie, dajc embedding.

Zbiorem danych bya kolekcja 400 milionw par (obraz, tekst), gdzie tekst opisywa zawarto obrazu. Nastpnie wykonano constrastive pre-training - model dostaje batch $N$ par (obraz, tekst) i ma przewidzie, ktre z $N \times N$ moliwych kombinacji (obraz, tekst) faktycznie wystpiy. W tym celu maksymalizuje podobiestwo cosinusowe par, ktre wystpiy w batchu, a minimalizuje podobiestwo dla $N^2 - N$ moliwych kombinacji, ktrych nie byo w batchu. W tym celu jest uywana entropia krzyowa z pewnymi modyfikacjami.

Wida wic, e model jest nienadzorowany, a zbir danych by ogromny, wic fine-tuning nie jest nam zasadniczo potrzebny. Mona go uy do klasyfikacji, ale tym nie bdziemy si zajmowa. Dodatkowo odpowiedni metryk odlegoci dla najbliszych ssiadw bdzie podobiestwo cosinusowe.

Biblioteka Sentence-Transformers powstaa dla rnych modeli transformerowych do zada nienadzorowanych, ale dodano do niej take bardzo wygodny interfejs dla modelu CLIP - przykad uycia.

Jako zbir wykorzystamy Amazon Berkeley Objects (ABO) Dataset - zbir obrazw produktw Amazona, stworzony we wsppracy z University of California, Berkeley. Wykorzystamy zminiaturyzowany zbir, w ktrym obrazy maj rozmiar 256x256 pikseli, bo na nasze potrzeby jest zupenie wystarczajcy.

from itertools import islice

import joblib
import torch
from sentence_transformers import SentenceTransformer
from tqdm.notebook import tqdm
from PIL import Image


MAX_IMAGES = 50000
BATCH_SIZE = joblib.cpu_count(only_physical_cores=True)


def vectorize_images(image_paths: pd.Series) -> np.ndarray:
    device = "cuda" if torch.cuda.is_available() else "cpu"
    model = SentenceTransformer("clip-ViT-B-32", device=device)

    # batch iterator, based on Python itertools example
    def batched(iterable, n: int):
        it = iter(iterable)
        while batch := tuple(islice(it, n)):
            yield batch

    # CLIP embeddings have 512 dimensions
    embeddings = np.empty((MAX_IMAGES, 512))

    # iterate with tqdm, it will give us a nice progress bar
    with tqdm(total=MAX_IMAGES) as pbar:
        start_idx = 0
        for batch in batched(image_paths.iloc[:MAX_IMAGES], BATCH_SIZE):
            end_idx = min(start_idx + len(batch), MAX_IMAGES)

            images = [Image.open(img_path) for img_path in batch]
            batch_embeddings = model.encode(images, batch_size=BATCH_SIZE)
            embeddings[start_idx:end_idx] = batch_embeddings
            start_idx = end_idx

            pbar.update(BATCH_SIZE)

    return embeddings

class ImageSearch:
    def __init__(self, image_paths: pd.Series, embeddings: np.ndarray):

        self.image_paths = image_paths.values

        device = "cuda" if torch.cuda.is_available() else "cpu"
        self.model = SentenceTransformer("clip-ViT-B-32", device=device)

        self.index = NNDescent(
            embeddings,
            metric="cosine",
            random_state=0,
            n_jobs=-1
        )

    def search(self, text: str, n_neighbors: int = 9) -> None:
        image_paths = self.query(text, n_neighbors)
        self.show(image_paths)

    def query(self, text: str, n_neighbors: int = 9) -> list[str]:
        text_embedding = self.model.encode([text])
        indices, _ = self.index.query(text_embedding, k=n_neighbors)
        return [self.image_paths[i] for i in indices[0]]

    def show(self, image_paths: list[str]) -> None:
        n = len(image_paths)
        ncols = int(np.ceil(np.sqrt(n)))
        nrows = int(np.ceil(n / ncols))

        fig = plt.figure(figsize=(10, 10))
        grid = ImageGrid(
            fig, 111,
            nrows_ncols=(nrows, ncols),
            axes_pad=0.1,
        )

        for ax, img_path in zip(grid, image_paths):
            try:
                img = Image.open(img_path)
                ax.imshow(img)
                ax.axis('off')
            except Exception as e:
                print(f"Error loading image {img_path}: {e}")
                ax.axis('off')

        for ax in grid[len(image_paths):]:
            ax.axis('off')

        plt.tight_layout()
        plt.show()

Zapoznaj si z algorytmem Condensed Nearest Neighbors . Jest to algorytm typu training set reduction, ktry zmniejsza rozmiar zbioru treningowego do klasyfikacji, usuwajc z niego "trywialne" prbki. Idea jest tutaj taka, e znaczenie maj zasadniczo tylko przykady blisko granicy decyzyjnej, a te daleko od niej s oczywiste i mona by tam ograniczy liczb prbek, przyspieszajc algorytmy najbliszych ssiadw.

Zaimplementuj algorytm Condensed Nearest Neighbors w postaci klasy kompatybilnej ze Scikit-learn, z metodami .fit() oraz .transform(). Wejciem powinny by macierz X oraz wektor y, a wyjciem zredukowane macierze, ograniczone tylko do interesujcych prbek. Niestety ten algorytm jest czuy na kolejno przetwarzania prbek, dlatego ma element losowy - zaimplementuj argument random_state w konstruktorze i go uyj do przemieszania losowo zbioru (mog si przyda generatory losowe z Numpy'a). Dodatkowo konstruktor powinien przyjmowa argument metric (analogicznie do KNeighborsClassifier) i wykorzystywa dan metryk jako sposb pomiaru odlegoci.

from sklearn.base import BaseEstimator, TransformerMixin
from sklearn.utils import check_random_state
import time
from sklearn.metrics import f1_score

class CondensedNearestNeighbor(TransformerMixin, BaseEstimator):
    def __init__(self, metric, random_state):
        self.random_state = random_state
        self.knn = KNeighborsClassifier(n_neighbors=1, metric=metric, n_jobs=-1)

    def fit(self, X, y):
        start_time = time.time()

        random_state = check_random_state(self.random_state)
        X, y = np.array(X), np.array(y)

        U = []
        for clas in np.unique(y):
            class_indices = np.where(y == clas)[0]
            U.append(random_state.choice(class_indices))

        U_X, U_y = X[U], y[U]
        self.knn.fit(U_X, U_y)

        while True:
            prev_size = len(U)
            indices = np.arange(len(X))
            random_state.shuffle(indices)

            for i in indices:
                if i not in U:

                  pred = self.knn.predict([X[i]])[0]

                  if pred != y[i]:
                      U_X = np.vstack([U_X, X[i]])
                      U_y = np.concatenate([U_y, [y[i]]])
                      U.append(i)
                      self.knn.fit(U_X, U_y)

            if len(U) == prev_size:
                break

        self.U_X_ = U_X
        self.U_y_ = U_y

        cnn_time = time.time() - start_time
        print(f"CNN execution time: {cnn_time:.4f} seconds")

        start_time = time.time()
        self.knn.fit(U_X, U_y)
        fit_time = time.time() - start_time
        print(f"Fitting time: {fit_time:.6f} seconds")
        return self

    def transform(self, X, y):
        return self.U_X_, self.U_y_

    def predict(self, X):
        self.knn.fit(self.U_X_, self.U_y_)
        return self.knn.predict(np.array(X))

    def predict_proba(self, X):
      self.knn.fit(self.U_X_, self.U_y_)
      return self.knn.predict_proba(np.array(X))
